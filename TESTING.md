# Тестирование: Домашняя библиотека

## Автоматические тесты

Запуск:

```bash
source venv/bin/activate
python manage.py test
```

Ожидаемый результат:

- 17 тестов пройдено.
- Ошибок нет.

### Покрытие тестами

| Сценарий | Файл | Проверка |
|----------|------|----------|
| Регистрация с уникальным username и email | `library/tests.py` | Создание пользователя; повторная регистрация с тем же email/username возвращает ошибку. |
| Вход с неверным паролем | `library/tests.py` | Отображается сообщение об ошибке в `non_field_errors`. |
| Создание книги | `library/tests.py` | Авторизованный пользователь может добавить книгу. |
| Выдача книги | `library/tests.py` | Можно взять свободную книгу; создаётся запись `Borrowing` с `returned_at=None`. |
| Возврат книги | `library/tests.py` | Возврат устанавливает `returned_at`. |
| Невозможно взять занятую книгу | `library/tests.py` | При активной выдаче POST на `borrow` возвращает ошибку или редирект без создания новой записи. |
| Ограничение срока возврата 365 дней | `library/tests.py` | Дата возврата через 380 дней отклоняется; 30 дней принимается. |
| Права доступа | `library/tests.py` | Только владелец может редактировать и удалять книгу; другой пользователь получает 403/404. |
| Month-picker формат | `library/tests.py` | Ввод `2026-08` преобразуется в `2026-08-01`; неверный формат отклоняется. |

## Ручной чек-лист

### Главная страница `/`

- [ ] Для неавторизованного пользователя отображается hero-секция и кнопки входа/регистрации.
- [ ] Для авторизованного пользователя отображается приветствие, статистика библиотеки и ссылки в каталог и dashboard.

### Регистрация `/register/`

- [ ] Успешная регистрация создаёт пользователя, выполняет автоматический вход и перенаправляет в каталог.
- [ ] Повторная регистрация с тем же `username` возвращает ошибку.
- [ ] Повторная регистрация с тем же `email` возвращает ошибку.
- [ ] Пароли хешируются (нельзя увидеть plaintext в БД).

### Вход `/login/`

- [ ] При неверных учётных данных отображается сообщение об ошибке.
- [ ] При правильных учётных данных происходит вход и редирект на главную.

### Каталог `/books/`

- [ ] Открытие каталога без параметров сортирует книги по названию от Я до А.
- [ ] Поиск по названию работает.
- [ ] Поиск по автору работает.
- [ ] Сортировка по автору работает в обоих направлениях.
- [ ] Сортировка по дате добавления работает в обоих направлениях.
- [ ] Query string сохраняет поиск и сортировку.
- [ ] Статус книги отображается корректно: свободна / у кого находится.

### Карточка книги `/books/<pk>/`

- [ ] Владелец видит кнопки «Редактировать» и «Удалить».
- [ ] Не владелец не видит кнопки редактирования и удаления.
- [ ] Свободная книга имеет кнопку «Взять почитать».
- [ ] Занятая книга не имеет кнопки «Взять почитать».
- [ ] Текущий читатель или владелец видит кнопку «Вернуть книгу».

### Выдача книги `/books/<pk>/borrow/`

- [ ] Форма содержит month-picker с placeholder `2026-08`.
- [ ] При выборе месяца и года день устанавливается в 1-е число.
- [ ] Срок возврата более 365 дней отклоняется.
- [ ] Прошедший срок отклоняется.
- [ ] После выдачи книга отмечается как занятая.

### Возврат книги `/books/<pk>/return/`

- [ ] Текущий читатель может вернуть книгу.
- [ ] Владелец может вернуть книгу.
- [ ] После возврата книга снова свободна.
- [ ] Запись о выдаче сохраняется в истории.

### Моя полка `/dashboard/`

- [ ] Отображаются книги, добавленные текущим пользователем.
- [ ] Отображаются книги, взятые текущим пользователем.
- [ ] Отображается история завершённых выдач.

### CRUD книги

- [ ] Авторизованный пользователь может добавить книгу.
- [ ] Владелец может отредактировать свою книгу.
- [ ] Владелец может удалить свою книгу.
- [ ] Другой пользователь не может отредактировать чужую книгу.
- [ ] Другой пользователь не может удалить чужую книгу.

### Дизайн и вёрстка

- [ ] Шрифт Inter загружается и применяется.
- [ ] Монохромная палитра, без ярких цветов и эмодзи.
- [ ] Header фиксированный с блюром.
- [ ] Кнопки и поля ввода имеют одинаковую высоту (44 px).
- [ ] Footer идёт естественно после контента, не прижат к низу экрана.
- [ ] Страница выглядит аккуратно на мобильном разрешении.


- [ ] Сервисы `web` и `nginx` запущены без ошибок.
- [ ] `http://localhost:8000/` открывается и возвращает 200.


- [ ] Образ собирается без ошибок.
