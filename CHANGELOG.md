# Changelog: Домашняя библиотека

## [1.1.0] — ревизия и исправления

### Исправления багов

- Исправлена инверсия статистики на главной странице: счётчик «На руках» теперь корректно показывает число выданных книг (ранее переменная `available_books` считала занятые книги, а шаблон вычитал её из общего количества, показывая свободные).
- Исправлен 500-ответ на карточке книги: `record.due_date.strftime(...)` вызывал `AttributeError` для записей истории без срока возврата; добавлена проверка `due_date is None`.
- Кнопки на страницах подтверждения удаления и возврата книги теперь используют собственные классы `.btn`/`.btn--primary`/`.btn--secondary` вместо несуществующих Pico.css-классов (`contrast`, `outline`), что нарушало монохромный дизайн и высоту 44px.
- Форма добавления/редактирования книги (`book_form.html`) переведена на `.form-card`/`.form-row`/`.btn`, как в остальных формах; `help_text` выводится через `|safe`.
- В `settings.py` добавлена переменная окружения `DB_PATH` с откатом к `/app/data/db.sqlite3` в контейнере и к `db.sqlite3` в корне проекта при локальной разработке; `runserver` теперь работает вне контейнера.
- На страницу регистрации добавлен вывод `form.non_field_errors()` (ранее отображались только ошибки полей).
- На модель `Borrowing` добавлено ограничение уникальности `UniqueConstraint("book_id", condition=Q(returned_at__isnull=True))`, гарантирующее одного активного читателя **на книгу** на уровне БД (миграция `0002`). Добавлен регрессионный тест `test_can_borrow_other_book_while_one_is_borrowed`.
- Поле `User.email` сделано уникальным на уровне БД (`unique=True`, миграция `0002`); ранее уникальность проверялась только формой регистрации.
- Исправлена ошибка валидации срока возврата: выбор текущего месяца больше не отклоняется как «в прошлом» (сравнение теперь ведётся с 1-м числом текущего месяца).
- Тест `test_cannot_borrow_already_borrowed` теперь отправляет дату в формате `YYYY-MM` (как ожидает форма), а не `YYYY-MM-DD`.
- Кнопка возврата на карточке книги переименована с «Отметить возврат» на «Вернуть книгу» (соответствует чек-листу `TESTING.md`).
- Пустой параметр `sort=` в каталоге больше не откатывается к `created_at`; используется документированное значение по умолчанию `title`.
- Обновлены устаревшие docstrings «Django 5.0.14» → «Django 5.2.15».
- Исправлен 500-ответ при гонке взятия книги: view `borrow_book` теперь обрабатывает `IntegrityError` от ограничения `unique_active_borrowing_per_book` (например, при двойном клике по «Взять почитать») и показывает «Эта книга уже занята.» вместо падения. Добавлен регрессионный тест `test_borrow_race_returns_redirect_not_500`.

### Добавлено

- Management-команда `seed_demo` для наполнения БД демонстрационными пользователями, книгами и выдачами (идемпотентна, безопасна к повторному запуску).

### Изменения интерфейса

- В каталоге кнопка взятия книги переименована с «Взять» на «Взять почитать» для единообразия с карточкой книги.
- В «Моей полке» для каждой книги добавлена кнопка «Удалить» рядом с «Редактировать».

### Документация

- В `REQUIREMENTS.md`, `TESTING.md`, `ADR.md` термин «Личная панель» заменён на «Моя полка» (соответствует фактическому интерфейсу).
- В `TESTING.md` уточнён шаг регистрации (автоматический вход и переход в каталог) и тест ограничения срока (380 дней вместо 400).
- В `ADR.md` исправлено внутренне противоречивое обоснование сортировки по умолчанию и описана переменная `DB_PATH`.

## [1.0.0] — текущая версия

### Основные решения

- Фреймворк — Django 5.2.
- Шаблонизатор — Jinja2 через встроенный бэкенд Django.
- База данных — SQLite.
- Стили — собственный CSS в монохромном editorial-минимализме.
- Контейнеризация — Podman / Docker с Gunicorn + Nginx + GHCR.
- Все пользователи равны; ролевая модель не используется.
- Язык интерфейса — русский.

### Сделано

#### Модели и бизнес-логика

- Модель `Book` с полями `title`, `author`, `isbn`, `description`, `owner`, `created_at`.
- Модель `Borrowing` для хранения истории выдач: `book`, `borrower`, `borrowed_at`, `returned_at`, `due_date`.
- Право редактировать и удалять книгу — только у владельца.
- Вернуть книгу может текущий читатель или владелец.

#### Аутентификация

- Регистрация с уникальными `username` и `email`.
- Хеширование паролей через PBKDF2 (Django по умолчанию).
- Вход и выход.
- Защищённые страницы требуют авторизации.
- Исправлено отображение `non_field_errors` на странице входа.

#### Каталог книг

- Поиск по названию и автору.
- Сортировка по названию, автору и дате добавления с направлением asc/desc.
- По умолчанию сортировка по названию, по убыванию (Я → А).
- Состояние поиска и сортировки сохраняется в query string.

#### Выдача и возврат

- Форма выдачи с month-picker (`type="month"`) и placeholder `2026-08`.
- Валидация формата `YYYY-MM`; день возврата устанавливается как 1-е число месяца.
- Ограничение срока возврата не более чем 365 дней от текущей даты.
- История всех выдач сохраняется в `Borrowing`.

#### Интерфейс

- Собственный CSS с шрифтом Inter.
- Монохромная палитра, тонкие границы, плоские кнопки.
- Фиксированный header с блюром.
- Footer идёт естественно после контента.
- Русский язык, без эмодзи.
- Кнопки и поля ввода одинаковой высоты.

#### Контейнеры и деплоймент

- `Containerfile`, `entrypoint.sh`, `podman-compose.yml`, `nginx.conf`.
- `.env.example`, `.dockerignore`.
- GitHub Actions workflow для сборки и публикации образа в GHCR.
- Volume для SQLite и volume для статических файлов.
- Инструкции по локальному запуску и production-деплою в `DEPLOYMENT.md`.

### Исправления багов

- Исправлена работа `url()` в Jinja2 для именованных аргументов (`pk=book.pk`).
- Исправлено отображение `help_text` в формах регистрации и входа через фильтр `|safe`.
- Исправлено отображение ошибок входа через `form.non_field_errors()`.

### Тесты

- Регистрация с уникальным `username` и `email`.
- Сообщение об ошибке при неудачном входе.
- Создание книги.
- Выдача и возврат книги.
- Блокировка взятия занятой книги.
- Ограничение срока возврата 365 днями.
- Права доступа: редактирование и удаление только владельцем.
- Формат month-picker `YYYY-MM` и подстановка 1-го числа.

### Известные ограничения

- Обложки книг и загрузка файлов не реализованы.
- Штрафы и уведомления за просрочку отсутствуют.
- Для production применяется upstream load balancer, TLS не обрабатывается внутри приложения.
