# Требования к проекту: Домашняя библиотека

## 1. Общее описание

Веб-приложение для учета домашней библиотеки. Зарегистрированные пользователи (друзья) могут просматривать доступные книги, брать их почитать и отмечать возврат. Владельцы книг могут добавлять, редактировать и удалять только свои книги. Приложение помогает не забывать, кому и что было отдано.

## 2. Источник требований

Задание на итоговую работу (файл `TASK.md`):

- Фреймворк на выбор: Flask или Django;
- Базовый функционал учета домашней библиотеки;
- Друзья смотрят доступные книги и берут их почитать;
- Владельцы не забывают, кому и что дали;
- Обязательное использование шаблонов Jinja;
- Подключение минимум одной базы данных SQLite или DuckDB;
- Аутентификация с возможностью регистрации, хешированием паролей и проверкой уже существующих пользователей.

## 3. Принятые архитектурные и технологические решения

| Аспект | Решение |
|--------|---------|
| Фреймворк | Django 5.2 |
| Шаблонизатор | Jinja2 через встроенный бэкенд Django |
| База данных | SQLite (единственная БД) |
| ORM | Django ORM |
| CSS | Собственный CSS на основе дизайн-системы проекта, шрифт Inter |
| Дизайн | Монохромный минимализм по мотивам проекта DOM-HTML: чёрно-белая палитра, тонкие границы, плоские кнопки, без эмодзи |
| WSGI-сервер | Gunicorn |
| Reverse proxy | Nginx |
| Контейнеризация | Podman / Docker с `podman-compose` |
| Реестр образов | GitHub Container Registry (GHCR) |
| Язык интерфейса | Русский |
| Хеширование паролей | PBKDF2 (Django по умолчанию) |
| Роли пользователей | Нет строгих ролей; все пользователи равны и считаются друзьями |
| История выдач | Хранится в модели `Borrowing` |
| Админка Django | Не используется |

## 4. Функциональные требования

### 4.1 Аутентификация

- Регистрация нового пользователя с полями: имя пользователя, email, пароль, подтверждение пароля.
- Проверка уникальности `username` и `email`.
- Хеширование паролей перед сохранением.
- Вход в систему по имени пользователя и паролю.
- После успешной регистрации выполняется автоматический вход и переход в каталог.
- Выход из системы.
- Доступ к защищенным страницам только для авторизованных пользователей.
- При неудачном входе отображается сообщение об ошибке.

### 4.2 Управление книгами

- Авторизованный пользователь может добавить книгу с полями: название, автор, ISBN, описание.
- Владелец книги может редактировать и удалять только свои книги.
- Любой авторизованный пользователь видит список всех книг.
- В списке книг отображается статус: свободна или у кого находится.

### 4.3 Поиск и сортировка книг

- Поиск по названию и автору через одно поле поиска.
- Сортировка по названию (А-Я / Я-А), автору (А-Я / Я-А) и дате добавления (новые / старые).
- Сортировка по умолчанию — по названию, по убыванию (Я → А).
- Состояние поиска и сортировки сохраняется в query string.

### 4.4 Выдача и возврат книг

- Пользователь может взять свободную книгу.
- При выдаче пользователь выбирает месяц и год возврата через нативный виджет month-picker.
- День возврата устанавливается как 1-е число выбранного месяца.
- Поле ввода содержит placeholder `2026-08` и подсказку формата `год-месяц`.
- Срок возврата не может превышать 365 дней от текущей даты.
- Пользователь может вернуть книгу, если он является текущим читателем или владельцем.
- Каждая выдача фиксируется в истории (`Borrowing`).
- В истории хранится: книга, читатель, дата выдачи, дата возврата (если вернули), срок возврата.

### 4.5 Моя полка (dashboard)

- Пользователь видит список книг, которые он добавил, с указанием текущего читателя.
- Пользователь видет список книг, которые он взял, с указанием срока возврата.
- Пользователь видит историю своих завершённых выдач.

### 4.6 Главная страница

- Для гостей: hero-секция с описанием приложения и кнопками входа/регистрации.
- Для авторизованных пользователей: приветствие, краткая статистика библиотеки и кнопки перехода в каталог и на «Мою полку».

## 5. Нефункциональные требования

- Каталог книг отображается в виде сетки карточек с тонкими границами и бейджами статуса.
- Минималистичный и информативный интерфейс в монохромном стиле, без эмодзи.
- Кнопки и поля ввода имеют одинаковую высоту.
- Фиксированный header с блюром и тонкой нижней границей.
- Footer идёт естественно после контента, не прижат к низу экрана.
- Русский язык интерфейса.
- Приложение должно работать в режиме разработки через `python manage.py runserver`.
- Приложение должно запускаться в контейнерах через `podman-compose`.
- Наличие тестов для моделей, форм и ключевых view.
- Возможность ручного деплоя на VPS с использованием образа из GHCR.

## 6. Ограничения и допущения

- Все зарегистрированные пользователи считаются друзьями и имеют доступ к каталогу.
- Email является уникальным и обязательным для регистрации.
- Владелец книги — это пользователь, который её создал.
- Просроченные книги отображаются, но штрафов и уведомлений не предусмотрено.
- Обложки книг и загрузка файлов не реализуются.
- Для production используется контейнеризация с Nginx и Gunicorn; TLS обрабатывается upstream load balancer.

## 7. Маршруты приложения

| URL | Назначение | Доступ |
|-----|------------|--------|
| `/` | Главная страница | Все |
| `/books/` | Каталог книг с поиском и сортировкой | Авторизованные |
| `/books/<pk>/` | Карточка книги с действиями | Авторизованные |
| `/books/add/` | Добавить книгу | Авторизованные |
| `/books/<pk>/edit/` | Редактировать книгу | Владелец |
| `/books/<pk>/delete/` | Удалить книгу | Владелец |
| `/books/<pk>/borrow/` | Взять книгу | Авторизованные, книга свободна |
| `/books/<pk>/return/` | Вернуть книгу | Читатель или владелец |
| `/dashboard/` | Моя полка | Авторизованные |
| `/register/` | Регистрация | Гости |
| `/login/` | Вход | Гости |
| `/logout/` | Выход | Авторизованные |

## 8. Модели данных

### `Book`

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | CharField(200) | Название книги |
| `author` | CharField(200, blank=True) | Автор |
| `isbn` | CharField(20, blank=True) | ISBN |
| `description` | TextField(blank=True) | Описание |
| `owner` | ForeignKey(User) | Владелец книги |
| `created_at` | DateTimeField(auto_now_add=True) | Дата добавления |

### `Borrowing`

| Поле | Тип | Описание |
|------|-----|----------|
| `book` | ForeignKey(Book) | Книга |
| `borrower` | ForeignKey(User) | Читатель |
| `borrowed_at` | DateTimeField(auto_now_add=True) | Дата выдачи |
| `returned_at` | DateTimeField(null=True, blank=True) | Дата возврата |
| `due_date` | DateField(null=True, blank=True) | Срок возврата |

## 9. Бизнес-правила

- Книга считается свободной, если у неё нет активной записи `Borrowing` (с `returned_at=None`).
- Одновременно только один пользователь может держать книгу.
- Срок возврата при выдаче не может быть более 365 дней от текущей даты.
- Редактирование и удаление книги доступны только владельцу.
- Вернуть книгу может текущий читатель или владелец книги.
- Повторная регистрация с существующим `username` или `email` невозможна.

## 10. Тестирование

- Тесты регистрации с проверкой уникальности `username` и `email`.
- Тесты сообщения об ошибке при неудачном входе.
- Тесты создания книги.
- Тесты выдачи и возврата книги.
- Тесты проверки невозможности взять занятую книгу.
- Тесты ограничения срока возврата (не более 365 дней).
- Тесты прав доступа: только владелец редактирует и удаляет книгу.
- Тесты month-picker формата `YYYY-MM` и подстановки 1-го числа.
